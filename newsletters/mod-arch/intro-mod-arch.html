A Developer's Guide to Modular Architecture

Your onboarding guide for contributing to a scalable, reusable, and consistent UI for Kubeflow.

Part 1: Engaging with the Community

Before writing code, integrate with the project's culture and communication channels.

💬 Join the Conversation

➤ Slack: The hub for real-time, informal communication, questions, and quick feedback.
➤ Community Calendar: Contains the schedule for all public meetings, including working group syncs.

📜 Governance & Contribution

➤ CLA: Signing the Contributor License Agreement is mandatory.
➤ Pull Requests: All code changes are submitted via PRs, following the guide in each repo.
➤ Standards: Adhere to project standards for code style and documentation.

Part 2: Navigating the Codebase

A map to the key repositories for any UI contributor.
Repository Relevance
kubeflow/community Governance, roadmaps, meeting notes.
kubeflow/manifests Manifests for installing Kubeflow.
kubeflow-ui-essentials CRITICAL: Shared React components, hooks, themes.
kubeflow/model-registry Upstream repo for the Model Registry feature.
kubeflow/notebooks Upstream repo for the Notebooks feature.

Part 3: Development & Deployment Modes

Work quickly locally, then validate against integrated environments before submitting.
Mode Use Case Benefit
Standalone Local UI Development Maximum speed, no cluster needed.
Integrated-Kubeflow Platform Integration Testing High fidelity, tests against real services.
Integrated-Federated Product-Level Integration Full validation for downstream products.

Part 4: The Migration Path for Existing Features

A 7-step process for refactoring legacy monolithic features into the new architecture.

➤ 1. Define API Contract: Audit interactions and create a formal OpenAPI spec for the new BFF.

➤ 2. Adopt Upstream-First Workflow: Integrate fully with the upstream project's community and processes.

➤ 3. Scaffold the New Module: Initialize a new repository using the standard modular template.

➤ 4. Build BFF Endpoints: Develop the skeleton of all API endpoints from the OpenAPI spec.

➤ 5. Port the Frontend UI: Systematically migrate the UI, utilizing the `kubeflow-ui-essentials` library.

➤ 6. Implement BFF Business Logic: Flesh out the BFF endpoints with business logic to handle data.

➤ 7. Integrate into Downstream: Once complete, integrate the new module into the main application.
